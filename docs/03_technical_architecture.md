# 03. Technical Architecture — стартовый черновик

## Общий подход

Проект строится вокруг простых серверных сервисов, клиентских контроллеров и общих модулей в `ReplicatedStorage`. На этапе MVP код должен оставаться безопасным, читаемым и минимальным, без внешних зависимостей и преждевременной оптимизации.

## Основные зоны ответственности

- `ReplicatedStorage/Modules` — общие константы и утилиты.
- `ReplicatedStorage/Remotes` — будущие RemoteEvent и RemoteFunction для связи клиента и сервера.
- `ServerScriptService` — точка входа сервера и игровые сервисы.
- `StarterPlayer/StarterPlayerScripts` — точка входа клиента и будущие клиентские контроллеры.
- `StarterGui` — интерфейсы Roblox Studio.
- `Workspace` — объекты мира и тестовые сцены.

## Будущие сервисы

### PlayerDataService

Отвечает за загрузку, хранение и сохранение данных игрока: валюта, прогресс квестов, состояние участка, уровень дома и базовый инвентарь.

### QuestService

Управляет состояниями квестов, выдачей задач, проверкой условий выполнения и наградами.

### PlotService

Отвечает за участки игроков, выдачу земли, состояние дома и уровни улучшений.

### CurrencyService

Работает с базовой валютой: начисление, списание и проверка баланса.

### InventoryService

Будущий сервис для ресурсов, предметов, инструментов и наград.

### NPCDialogueService

Будущий сервис для диалогов NPC, веток реплик и связки диалогов с квестами.

### CombatService

Будущий сервис для PvE/PvP-боёв, урона, наград за победы и проверки безопасных зон.

## Минимальное ядро первой версии

В первой версии реализуем только минимальное ядро:

- `PlayerDataService`;
- `QuestService`;
- `PlotService`;
- `CurrencyService`.

Остальные сервисы описываются в документации, но не реализуются до появления реальной необходимости в MVP.

## Принципы MVP-кода

- Простые ModuleScript-заглушки с понятными интерфейсами.
- Сервер является источником истины для прогресса, валюты и владений.
- Клиент отвечает только за отображение и пользовательский ввод.
- Никаких внешних зависимостей на старте.
- Комментарии на русском языке для быстрой ориентации команды.
