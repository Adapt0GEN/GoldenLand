# 05. Current State - GoldenLand

## Идентификация проекта

- Рабочее название: **«Златоземье: Своя Земля»**.
- Roblox-проект: **GoldenLand**.
- Текущий рабочий стек: **Codex Desktop + Rojo + Roblox Studio**.
- Исходный код живет в папке `src/`.
- Roblox Studio получает код через Rojo.

## Важные ограничения окружения

- `Workspace` не синхронизируется через Rojo.
- `Baseplate` и `SpawnLocation` остаются внутри `.rbxl`.
- Rojo-конфиг не должен мапить `Workspace`.
- Нельзя допускать дублей `ServerMain`, `Services`, `StarterPlayerScripts`.
- Игровую карту и объекты `Workspace` можно править в Roblox Studio, но кодовые сервисы правятся в `src`.

## Текущий рабочий игровой цикл

1. Игрок появляется в стартовой зоне.
2. Игрок говорит со старостой и получает стартовый квест `first_steps`.
3. Игрок добывает базовые ресурсы через `ProximityPrompt`.
4. UI ресурсов показывает текущие значения:
   - `Золото`;
   - `Дерево`;
   - `Камень`;
   - `Металл`;
   - `Дом: уровень N`;
   - `Инструменты: уровень N`.
5. После выполнения стартового квеста открывается личный участок.
6. На участке работают дом, склад и мастерская.
7. Игрок улучшает дом за ресурсы.
8. Игрок использует склад как часть базовой инфраструктуры участка.
9. Игрок создает в мастерской `Набор инструментов I`, повышает `ToolKitLevel` и открывает ForestZone.
10. В ForestZone игрок добывает расширенные ресурсы и очищает зону до состояния `Empty`.
11. Игрок создает `Набор инструментов II`, повышает `ToolKitLevel` до уровня, достаточного для входа в RockZone.
12. При `ToolKitLevel >= 2` открывается RockZone, а флаг `RockZoneUnlocked` сохраняется в профиле.
13. В RockZone появляются точки добычи камня и металла: `RichStoneNode` и `MetalVein`.
14. После добычи UI ресурсов обновляется, а точки RockZone уходят на cooldown и затем respawn.
15. Прогресс игрока, открытые зоны и состояние очищенной ForestZone сохраняются между игровыми сессиями.

Текущий экономический контур доведен до добычи камня и металла в RockZone: игрок проходит стартовую экономику, развивает участок, делает инструменты I и II, открывает ForestZone и RockZone, затем получает доступ к более ценным источникам `Stone` и `Metal`.

## Ресурсы

Сейчас в игре работают базовые ресурсы:

```text
Gold
Wood
Stone
Metal
```

Игрок может добывать стартовые ресурсы, ресурсы ForestZone и ресурсы RockZone. `CurrencyService` отвечает за выдачу ресурсов, обновление данных игрока и отображение значений в UI. Ресурсы используются для базового развития участка, включая улучшение дома и создание инструментов в мастерской.

## Дом, склад и мастерская

### Дом

- Дом создается на личном участке игрока.
- Дом имеет уровень `HouseLevel`.
- Уровень дома отображается в UI как `Дом: уровень N`.
- Улучшение дома выполняется через `ProximityPrompt` за ресурсы.
- `HouseLevel` сохраняется через данные игрока.

### Склад

- Склад реализован как часть инфраструктуры личного участка.
- Склад работает вместе с текущей ресурсной экономикой.
- На текущем этапе склад не требует отдельного расширения механик.

### Мастерская

- Мастерская реализована на личном участке.
- В мастерской создаются `Набор инструментов I` и `Набор инструментов II`.
- Мастерская связана с прогрессией `ToolKitLevel`.
- Первый набор инструментов открывает ForestZone.
- Второй набор инструментов открывает RockZone.

## ToolKitLevel и инструменты

- `ToolKitLevel` реализован и сохраняется в данных игрока.
- UI отображает уровень инструментов как `Инструменты: уровень N`.
- `Набор инструментов I` создается в мастерской и повышает `ToolKitLevel` до уровня, достаточного для входа в ForestZone.
- `Набор инструментов II` создается в мастерской и повышает `ToolKitLevel` до уровня, достаточного для входа в RockZone.

Условия открытия зон:

```text
ForestZone: ToolKitLevel >= 1
RockZone: ToolKitLevel >= 2
```

## ForestZone

ForestZone завершена как первый расширенный участок прогрессии после стартовой базы.

Сейчас работает:

- открытие прохода в ForestZone при `ToolKitLevel >= 1`;
- сохранение `ForestUnlocked`;
- сохранение `ForestZoneState`;
- сохранение состояния `ForestArea_01`;
- добыча объектов внутри лесной зоны;
- визуальные стадии лесной зоны:
  - `Active`;
  - `Medium`;
  - `Low`;
  - `Empty`;
- удаление старых `ForestZone` visual state objects перед созданием нового визуального состояния;
- создание `Empty`-визуала с 0 деревьев;
- восстановление `Empty`-визуала после перезапуска Play;
- отсутствие повторного наполнения очищенной ForestZone активными ресурсами.

`ForestZoneState` хранит общее состояние лесной зоны. Когда ресурсы зоны полностью выработаны, `ForestZoneState` становится `Empty` и сохраняется. После перезапуска Play зона не возвращается в густой лес, а визуально восстанавливается как очищенная.

`ForestArea_01` хранит состояние конкретной области внутри ForestZone. При полной выработке ресурсов область становится `Empty`, а состояние сохраняется между запусками Play.

## Объекты ForestZone

### ForestTreeCluster

- Имеет `RemainingActions`.
- При добыче `RemainingActions` уменьшается.
- Когда `RemainingActions = 0`, `ForestTreeCluster` становится `Empty`.
- После выработки кластера прогресс учитывается в состоянии `ForestArea_01` и `ForestZoneState`.

### ForestStone_01 и ForestStone_02

- Имеют поле `State`.
- Их состояние сохраняется как часть прогресса лесной зоны.
- Они участвуют в переходе `ForestArea_01` к состоянию `Empty`.

## RockZone

RockZone реализована как следующий слой добычи после ForestZone.

Сейчас работает:

- открытие RockZone при `ToolKitLevel >= 2`;
- сохранение `RockZoneUnlocked`;
- создание ресурсных точек только при `RockZoneUnlocked == true`;
- защита от дублей ресурсных точек при повторных вызовах создания зоны;
- добыча ресурсов через `ProximityPrompt`;
- обновление UI после добычи;
- cooldown/respawn ресурсных точек RockZone.

RockZone не появляется как активная ресурсная зона для игроков, которые еще не открыли ее через прогрессию инструментов. Это сохраняет линейный экономический контур: сначала стартовые ресурсы, затем ForestZone, затем RockZone.

## Объекты RockZone

### RichStoneNode

- Богатый камень в RockZone.
- При добыче дает:

```text
+4 Stone
```

- Появляется только если `RockZoneUnlocked == true`.
- Не создает дублей при повторном создании/обновлении зоны.
- После добычи уходит на cooldown и возвращается через respawn.

### MetalVein

- Металлическая жила в RockZone.
- При добыче дает:

```text
+3 Metal
```

- Появляется только если `RockZoneUnlocked == true`.
- Не создает дублей при повторном создании/обновлении зоны.
- После добычи уходит на cooldown и возвращается через respawn.

## Admin tool

Admin tool работает и используется как вспомогательный инструмент для проверки прогресса MVP:

- помогает быстро тестировать игровые состояния;
- позволяет ускорять проверку ресурсов и прогрессии;
- используется для валидации открытия зон и поведения экономического контура;
- не является частью обычного пользовательского игрового цикла.

## Action Preview UI

Action Preview UI работает и показывает игроку предварительную информацию по доступным действиям.

Сейчас он поддерживает понятный цикл взаимодействия:

- игрок подходит к интерактивному объекту;
- UI показывает ожидаемое действие;
- игрок заранее видит, что произойдет при использовании `ProximityPrompt`;
- после выполнения действия ресурсный UI обновляется актуальными значениями.

Action Preview UI особенно важен для добычи, потому что помогает отличать обычные объекты от полезных ресурсных точек и делает поведение ForestZone/RockZone читаемым.

## Текущие работающие системы

- `PlayerDataService` - профиль игрока, загрузка/сохранение через DataStore, публичные данные для UI.
- `CurrencyService` - выдача ресурсов игроку и обновление UI ресурсов.
- `QuestService` - квест `first_steps`, прогресс сбора дерева, награды и UI квеста.
- `NPCService` - NPC `VillageElder`, выдача и сдача стартового квеста.
- `ResourceService` - стартовые ресурсы с `ProximityPrompt`, ресурсные точки RockZone и их cooldown/respawn.
- `PlotService` - личный участок, дом, склад, мастерская, визуальные уровни дома и улучшение дома за ресурсы.
- `WorldService` - стартовая сцена, зоны старосты, ресурсов, указатель к личной земле, ForestZone, RockZone и визуальные состояния зон.
- `ClientMain UI` - клиентский UI квеста, ресурсов, уровня дома, уровня инструментов, коротких сообщений и Action Preview UI.
