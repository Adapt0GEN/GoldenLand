# 05. Current State — GoldenLand

## Идентификация проекта

- Рабочее название: **«Златоземье: Своя Земля»**.
- Roblox-проект: **GoldenLand**.
- Текущий рабочий стек: **Codex Desktop + Rojo + Roblox Studio**.
- Исходный код живёт в папке `src/`.
- Roblox Studio получает код через Rojo.

## Важные ограничения окружения

- `Workspace` не синхронизируется через Rojo.
- `Baseplate` и `SpawnLocation` остаются внутри `.rbxl`.
- Rojo-конфиг не должен мапить `Workspace`.
- Нельзя допускать дублей `ServerMain`, `Services`, `StarterPlayerScripts`.
- Игровую карту и объекты `Workspace` можно править в Roblox Studio, но кодовые сервисы правятся в `src`.

## Текущие работающие системы

- `PlayerDataService` — профиль игрока, загрузка/сохранение через DataStore, публичные данные для UI.
- `CurrencyService` — выдача ресурсов игроку и обновление UI ресурсов.
- `QuestService` — квест `first_steps`, прогресс сбора дерева, награды и UI квеста.
- `NPCService` — NPC `VillageElder`, выдача и сдача стартового квеста.
- `ResourceService` — деревья с `ProximityPrompt` для сбора дерева.
- `PlotService` — личный участок, дом, визуальные уровни дома, улучшение дома за ресурсы.
- `WorldService` — стартовая сцена, зоны старосты, ресурсов и указатель к личной земле.
- `ClientMain UI` — клиентский UI квеста, ресурсов, уровня дома и коротких сообщений.

## Текущий игровой цикл

1. Игрок появляется в стартовой зоне.
2. Игрок говорит со старостой.
3. Староста выдаёт квест `first_steps`.
4. Игрок собирает 3 дерева через `ProximityPrompt`.
5. UI показывает прогресс `Собери дерево: X/3`.
6. Игрок возвращается к старосте.
7. Староста завершает квест.
8. Игрок получает награду.
9. Открывается личный участок.
10. Дом создаётся на участке.
11. Дом улучшается через `ProximityPrompt` за ресурсы.
12. `HouseLevel` сохраняется через DataStore.

## Текущий результат MVP

На текущем этапе проект уже имеет рабочий первый вертикальный срез: стартовая сцена, NPC, квест, сбор ресурсов, UI, личная земля, дом, улучшение дома и сохранение прогресса игрока.
