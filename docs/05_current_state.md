# 05. Current State - GoldenLand

## Идентификация проекта

- Рабочее название: **«Златоземье: Своя Земля»**.
- Roblox-проект: **GoldenLand**.
- Текущий рабочий стек: **Codex Desktop + Rojo + Roblox Studio**.
- Исходный код живёт в папке `src/`.
- Roblox Studio получает код через Rojo.

## Важные ограничения окружения

- `Workspace` не синхронизируется через Rojo.
- `Baseplate` и `SpawnLocation` остаются внутри `.rbxl`.
- Rojo-конфиг не должен мапить `Workspace`.
- Нельзя допускать дублей `ServerMain`, `Services`, `StarterPlayerScripts`.
- Игровую карту и объекты `Workspace` можно править в Roblox Studio, но кодовые сервисы правятся в `src`.

## Текущий рабочий игровой цикл

1. Игрок появляется в стартовой зоне.
2. Игрок говорит со старостой и получает стартовый квест `first_steps`.
3. Игрок добывает стартовые ресурсы через `ProximityPrompt`.
4. UI ресурсов показывает текущие значения:
   - `Золото`;
   - `Дерево`;
   - `Камень`;
   - `Металл`;
   - `Дом: уровень N`;
   - `Инструменты: уровень N`.
5. После выполнения стартового квеста открывается личный участок.
6. На участке работает дом, склад и мастерская.
7. Игрок улучшает дом за ресурсы.
8. Игрок использует склад как часть базовой инфраструктуры участка.
9. Игрок создаёт в мастерской `Набор инструментов I`.
10. После создания набора инструментов повышается `ToolKitLevel`.
11. При `ToolKitLevel >= 1` открывается проход в лесную зону.
12. Игрок очищает лесную зону от ресурсов.
13. Состояние прогресса игрока и состояние лесной зоны сохраняются.
14. После перезапуска Play уже очищенная лесная зона восстанавливается в состоянии `Empty`.

## Ресурсы

Сейчас в игре работают базовые ресурсы:

```text
Gold
Wood
Stone
Metal
```

Игрок может добывать стартовые ресурсы. `CurrencyService` отвечает за выдачу ресурсов, обновление данных игрока и отображение значений в UI. Ресурсы используются для базового развития участка, включая улучшение дома и создание инструментов в мастерской.

## Дом, склад и мастерская

### Дом

- Дом создаётся на личном участке игрока.
- Дом имеет уровень `HouseLevel`.
- Уровень дома отображается в UI как `Дом: уровень N`.
- Улучшение дома выполняется через `ProximityPrompt` за ресурсы.
- `HouseLevel` сохраняется через данные игрока.

### Склад

- Склад реализован как часть инфраструктуры личного участка.
- Склад работает вместе с текущей ресурсной экономикой.
- На текущем этапе склад не требует отдельного расширения механик.

### Мастерская

- Мастерская реализована на личном участке.
- В мастерской создаётся `Набор инструментов I`.
- Мастерская связана с прогрессией `ToolKitLevel`.
- Создание первого набора инструментов открывает следующий слой прогресса: доступ к лесной зоне.

## ToolKitLevel и Набор инструментов I

- `ToolKitLevel` реализован и сохраняется в данных игрока.
- UI отображает уровень инструментов как `Инструменты: уровень N`.
- `Набор инструментов I` создаётся в мастерской.
- После создания `Набор инструментов I` повышает `ToolKitLevel` до значения, достаточного для входа в ForestZone.
- Проход в лесную зону открывается при условии:

```text
ToolKitLevel >= 1
```

## ForestZone

Лесная зона завершена как первый расширенный участок прогрессии после стартовой базы.

Сейчас работает:

- открытие прохода в ForestZone при `ToolKitLevel >= 1`;
- сохранение `ForestUnlocked`;
- сохранение `ForestZoneState`;
- сохранение состояния `ForestArea_01`;
- добыча объектов внутри лесной зоны;
- визуальные стадии лесной зоны:
  - `Active`;
  - `Medium`;
  - `Low`;
  - `Empty`;
- удаление старых `ForestZone` VisualStateObjects перед созданием нового визуального состояния;
- создание `Empty`-визуала с 0 деревьев;
- восстановление `Empty`-визуала после перезапуска Play.

## ForestZoneState и ForestArea_01

### ForestZoneState

`ForestZoneState` хранит общее состояние лесной зоны.

Текущее важное состояние:

```text
Empty
```

Когда ресурсы зоны полностью выработаны, `ForestZoneState` становится `Empty` и сохраняется. После перезапуска Play зона не возвращается в густой лес, а визуально восстанавливается как очищенная.

### ForestArea_01

`ForestArea_01` хранит состояние конкретной области внутри ForestZone.

Сейчас реализовано:

- `ForestArea_01` сохраняет своё состояние;
- при полной выработке ресурсов область становится `Empty`;
- состояние `Empty` сохраняется между запусками Play;
- временный DEBUG reset для `ForestArea_01` убран.

## Объекты ForestZone

### ForestTreeCluster

- Имеет `RemainingActions`.
- При добыче `RemainingActions` уменьшается.
- Когда `RemainingActions = 0`, `ForestTreeCluster` становится `Empty`.
- После выработки кластера прогресс учитывается в состоянии `ForestArea_01` и `ForestZoneState`.

### ForestStone_01 и ForestStone_02

- Имеют поле `State`.
- Их состояние сохраняется как часть прогресса лесной зоны.
- Они участвуют в переходе ForestArea_01 к состоянию `Empty`.

## Empty-состояние ForestZone

Полностью очищенная ForestZone теперь является устойчивым состоянием, а не временным визуальным эффектом.

Когда ForestZone переходит в `Empty`:

- `ForestTreeCluster` уже не восстанавливается как активный ресурс;
- `ForestArea_01` становится `Empty`;
- `ForestZoneState` становится `Empty`;
- старые ForestZone VisualStateObjects удаляются;
- создаётся отдельный `Empty`-визуал;
- `Empty`-визуал создаётся с 0 деревьев;
- после перезапуска Play зона остаётся `Empty`.

Это подтверждает, что действия игрока меняют мир и результат сохраняется между игровыми сессиями.

## Текущие работающие системы

- `PlayerDataService` - профиль игрока, загрузка/сохранение через DataStore, публичные данные для UI.
- `CurrencyService` - выдача ресурсов игроку и обновление UI ресурсов.
- `QuestService` - квест `first_steps`, прогресс сбора дерева, награды и UI квеста.
- `NPCService` - NPC `VillageElder`, выдача и сдача стартового квеста.
- `ResourceService` - стартовые ресурсы с `ProximityPrompt`.
- `PlotService` - личный участок, дом, склад, мастерская, визуальные уровни дома и улучшение дома за ресурсы.
- `WorldService` - стартовая сцена, зоны старосты, ресурсов, указатель к личной земле, ForestZone и визуальные состояния лесной зоны.
- `ClientMain UI` - клиентский UI квеста, ресурсов, уровня дома, уровня инструментов и коротких сообщений.
