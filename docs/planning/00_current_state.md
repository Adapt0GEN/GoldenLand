# 00. Текущее состояние проекта

Проект: GoldenLand / «Златоземье: Своя Земля»

---

## 1. Реализованная основа

В проекте уже заложена стартовая петля:

```text
личная земля
↓
сбор ресурсов
↓
дом / склад / мастерская
↓
металл
↓
Набор инструментов I
↓
расчистка прохода
↓
ForestZone
```

Базовые ресурсы:

```text
Дерево
Камень
Металл
Золото
```

Роли:

```text
Дерево — строительство, мосты, укрепления, инструменты.
Камень — фундамент, стены, печи, дороги, укрепления.
Металл — инструменты, механизмы, оружие, усиленные постройки.
Золото — ценность, торговля, особые улучшения, редкие постройки.
```

---

## 2. Важные технические решения

```text
ResourceService — обычная повторяемая добыча ресурсов.
QuestService — прогресс квестов.
CurrencyService — изменение Gold/Wood/Stone/Metal.
PlayerDataService — профиль, сохранение и обновление UI.
WorldService — создание мира, зон и объектов освоения.
```

Обычная добыча ресурсов не должна зависеть от квеста `first_steps`.

---

## 3. Набор инструментов I

Набор инструментов I — не одноразовый квестовый ключ, а технологический уровень лагеря.

```lua
ToolKitLevel = 0 -- инструментов нет
ToolKitLevel = 1 -- Набор инструментов I
```

Он используется для расширения территории и будущих действий:

```text
расчистка проходов
разбор завалов
ремонт мостов
открытие новых ресурсных точек
доступ к новым зонам
```

---

## 4. ForestZone

Текущая модель:

```lua
ForestUnlocked = false
ForestZoneState = "Locked"
ForestZoneClearedObjects = {}
```

Состояния MVP:

```text
Locked — зона закрыта
Active — зона открыта и содержит объекты освоения
Empty — объекты освоения зоны очищены
```

Будущие состояния:

```text
Overgrown — зона открыта, но выглядит заросшей
Cleared — зона визуально освоена / преобразована
```

---

## 5. Разделение объектов зоны

```text
DecorOnly — только визуал
GlobalResourceNode — обычная ресурсная точка с респавном
ZoneClearObject — объект освоения зоны, очищается один раз
ZoneResourceNode — ресурс зоны, может респавниться, если так задумано
ZoneOneTimeReward — одноразовая находка или награда
```

Главное правило:

```text
Обычный ресурс:
добывается → временно исчезает → респавнится

Объект освоения зоны:
очищается → сохраняется как очищенный → не респавнится
```

## 6. Текущая проблема после MVP 0.2.6

Тест показал частичный успех:

```text
ForestZoneState стал Empty
WorldService рендерит Empty
табличка "Лесная зона очищена" появилась
```

Но проблема осталась:

```text
лесная зона визуально не очищается полностью
в Empty остаются лишние деревья / объекты
в Output есть [ResourceService] Created ForestZone resources
```

Вывод:

```text
WorldService создаёт Empty-визуал,
но ResourceService всё ещё создаёт ресурсы/деревья внутри ForestZone.
```

Текущая ближайшая задача — исправить конфликт и добиться полной визуальной очистки Empty.
