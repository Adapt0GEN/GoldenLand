# 00. Текущее состояние проекта

Дата фиксации: 2026-05-19

Проект: GoldenLand / «Златоземье: Своя Земля»

---

## 1. Общая концепция

Игра строится вокруг постепенного освоения личной земли / острова.

Базовая петля:

```text
стартовая зона
↓
сбор ресурсов
↓
постройки лагеря
↓
технологический прогресс
↓
открытие новых зон
↓
освоение зоны
↓
новые ресурсы, угрозы, жители и постройки
↓
дальнейшая экспансия острова
```

Ключевой принцип: игрок должен видеть, что его действия меняют мир.

---

## 2. Базовые ресурсы

Базовая ресурсная четвёрка:

```text
Дерево
Камень
Металл
Золото
```

Роли:

```text
Дерево — строительство, мосты, укрепления, инструменты.
Камень — фундамент, стены, печи, дороги, укрепления.
Металл — инструменты, механизмы, оружие, усиленные постройки.
Золото — ценность, торговля, особые улучшения, редкие постройки.
```

---

## 3. Сервисы и ответственность

Принято разделение:

```text
ResourceService — обычная повторяемая добыча ресурсов.
QuestService — прогресс квестов.
CurrencyService — изменение Gold/Wood/Stone/Metal.
PlayerDataService — профиль, сохранение и обновление UI.
WorldService — создание мира, зон и объектов освоения.
```

Обычная добыча ресурсов не должна зависеть от квеста `first_steps`.


## 5. ForestZone

Логика открытия:

```text
ToolKitLevel >= 1
↓
игрок расчищает BlockedPathToForest
↓
ForestUnlocked = true
↓
открывается ForestZone
```

Базовые поля профиля:

```lua
ForestUnlocked = false
ToolKitLevel = 0
```

После открытия:

```lua
ForestUnlocked = true
```

## 6. Последние выводы по зонам и ресурсам

После тестов стало ясно: ForestZone нельзя проектировать как обычный набор респавнящихся ResourceNode.

Нужно разделить:

```text
1. Обычные ресурсные точки.
2. Объекты освоения зоны.
3. Декор.
4. Одноразовые награды.
5. Настоящие ресурсы зоны, если они должны респавниться.
```

### Обычный ресурс

Поведение:

```text
добывается
даёт ресурс
временно исчезает
респавнится по cooldown
```

Пример:

```text
стартовое дерево
стартовый камень
металлическая руда стартовой зоны
```

### Объект освоения зоны

Поведение:

```text
очищается один раз
может дать ресурс
objectId сохраняется в профиле
не респавнится
влияет на состояние зоны
```

Пример:

```text
завал в ForestZone
старое дерево
каменная куча
густые заросли
```

## 7. Текущая проблема ForestZone

По тестам:

```text
с зоной есть взаимодействие
добыча материалов работает
объекты исчезают за одно действие
часть объектов появляется снова
декоративные деревья остаются после Empty
камни респавнятся по cooldown
```

Главный вывод:

```text
ForestZone сейчас смешивает декор, обычные ресурсы и объекты освоения.
```

## 8. Новая целевая модель ForestZone

Минимальная модель состояний:

```text
Locked — зона закрыта
Overgrown — зона открыта, но заросшая
Active — зона содержит интерактивные объекты освоения
Empty — интерактивные объекты зоны очищены
Cleared — зона визуально освоена / преобразована
```

Для MVP можно начать с:

```text
Locked
Active
Empty
```

Поля профиля:

```lua
ForestUnlocked = false
ForestZoneState = "Locked"
ForestZoneClearedObjects = {}
```
