# MVP 0.2.x — Технологический прогресс и ForestZone

Цель:

```text
стартовые ресурсы
↓
металл
↓
мастерская
↓
Набор инструментов I
↓
расчистка прохода
↓
ForestZone
↓
освоение ForestZone
```

## 0.2.1 — Металл

```text
Metal
MetalNode
добыча металла
UI
сохранение
```

## 0.2.2 — Набор инструментов I

```text
ToolKitLevel = 1
```

Набор инструментов не исчезает после применения.

## 0.2.3 — Подготовка к расширению территории

Проверить:

```text
ToolKitLevel сохраняется
UI не ломается
мастерская работает
ресурсы списываются корректно
```

## 0.2.4 — Расчистка прохода в лес

```text
BlockedPathToForest
ClearForestPathPrompt
ForestUnlocked
ForestZone
```

## 0.2.5 — Архитектура зон и ресурсы внутри зон

Ближайший важный этап.

Цель: ForestZone не должна быть набором обычных респавнящихся ресурсов.

Нужно ввести:

```text
ForestZoneState
ForestZoneClearedObjects
DecorOnly
ZoneClearObject
ZoneResourceNode
ZoneOneTimeReward
разделение decor / interactives / resources
```

## 0.2.6 — Визуальное изменение ForestZone

После действий игрока зона должна меняться:

```text
заросшая
частично очищенная
пустая
освоенная
```

## 0.2.7 — Первые уникальные ресурсы леса

Добавить лесные ресурсы, заранее определив тип поведения:

```text
респавнится
очищается один раз
является одноразовой находкой
является декором
```

Примеры:

```text
лоза
смола
грибы
травы
редкая древесина
```

## 0.2.8 — Первый выход из ForestZone в следующую область

Возможные переходы:

```text
старый мост
болотный проход
заваленная тропа
пещерный вход
```
