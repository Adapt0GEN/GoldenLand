# DEC-001 — Ресурсы должны быть базовой четвёркой

## Решение

```text
Дерево

```

Для игры про освоение острова нужны разные роли ресурсов.

# DEC-003 — Добыча ресурсов не должна зависеть от квеста

## Решение

Ресурсы должны добываться всегда, даже если квест `first_steps` завершён.

## Разделение ответственности

```text
ResourceService — обычная добыча ресурсов
QuestService — прогресс квеста
CurrencyService — изменение ресурсов
PlayerDataService — профиль, сохранение, UI update
WorldService — состояние мира и зон
```

# DEC-007 — Разделить обычные ресурсы и объекты освоения зоны

## Решение

Вводим различие:

```text
GlobalResourceNode
ZoneClearObject
ZoneResourceNode
ZoneOneTimeReward
DecorOnly
```

## Причина

ForestZone не должна респавнить всё подряд.

Если игрок очищает завал или старое дерево в зоне, это часть освоения территории, а не бесконечный фарм.

## Последствие

Объекты освоения зоны должны:

```text
иметь objectId
сохраняться в ForestZoneClearedObjects
не появляться после очистки
участвовать в расчёте ForestZoneState
```

# DEC-008 — У каждой зоны должен быть паспорт

## Решение

Каждая зона должна описываться через набор правил:

```text
ZoneId
UnlockCondition
StateField
ClearedObjectsField
ObjectTypes
VisualStates
ResourceRules
ExitRules
```

ForestZone становится первым шаблоном для будущих зон.

---

# DEC-009 — Empty не должен выглядеть как нетронутый лес

## Решение

Если ForestZone перешла в Empty, визуал должен измениться.

Недопустимо:

```text
зона считается Empty
но в ней остались густые декоративные деревья
```
