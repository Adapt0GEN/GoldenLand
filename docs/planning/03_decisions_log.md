# DEC-004 — ForestZone должна быть зоной с состоянием

## Решение

ForestZone не должна быть просто набором респавнящихся объектов.

## Модель MVP

```text
Locked
Active
Empty
```

## Будущая модель

```text
Locked
Overgrown
Active
Empty
Cleared
```

# DEC-009 — Empty не должен выглядеть как нетронутый лес

## Решение

Если ForestZone перешла в Empty, визуал должен измениться.

Недопустимо:

```text
зона считается Empty
но в ней остались густые декоративные деревья
```

# DEC-010 — Визуальное развитие зоны должно идти после логики состояния

## Решение

Сначала реализуются состояние и сохранение зоны, потом визуальное развитие.

## Причина

Если сначала делать красивый визуал без стабильной логики, зона снова начнёт ломаться при перезапуске, очистке или повторном входе.

## Последствие

MVP 0.2.5 закрывает логику.
MVP 0.2.6 улучшает визуальное состояние.
