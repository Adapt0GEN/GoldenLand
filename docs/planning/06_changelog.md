# 2026-05-31 — Фикс 1.2 (архитектурный): привязка к земле через raycast

Корневая причина «висящих» моделей: проект хардкодит Y, предполагая землю на Y=0,
а на реальной карте поверхность не на нуле.

```text
- добавлен getGroundY(x, z): луч сверху вниз находит реальную высоту земли
  (работает с baseplate, террейном и склонами)
- враги и структуры лагеря теперь ставятся на найденную землю, без хардкода Y
- на будущее: тот же подход стоит применить к остальным сервисам мира
```

# 2026-05-31 — Фаза 1, шаг 1.1: основа боя (меч + hitbox + манекен)

Решения по Фазе 1: бой — оружие-Tool + серверный hitbox (риг R15), охват — соло.

Сделано в коде (новый `CombatService`):

```text
- меч-Tool выдаётся игроку в Backpack при спавне (без внешних ассетов)
- бой серверный: Tool.Activated → hitbox по передней дуге → урон врагам
- враг-манекен TrainingDummy с собственной системой HP (атрибуты + полоска BillboardGui)
- при смерти врага: награда Gold атакующему + респавн манекена через 5 сек
- визуальный эффект замаха перед игроком
- CombatService.Start() подключён в ServerMain
```

Требует проверки в Roblox Studio: экипировка меча, нанесение урона по дуге,
смерть/респавн манекена, начисление Gold. Анимация замаха R15 — отдельный
шаг полировки (нужен загруженный assetId анимации).
