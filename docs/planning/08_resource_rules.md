# 08. Resource Rules

Этот файл фиксирует правила ресурсов в проекте.

---

## 1. Главный принцип

Не все объекты, которые дают ресурсы, являются обычными ресурсными точками.

Нужно различать:

```text
обычный фарм
освоение зоны
одноразовую награду
декор
```

---

## 2. Обычные ресурсы

Обычные ресурсы — это повторяемая добыча.

Примеры:

```text
дерево в стартовой зоне
камень в стартовой зоне
металл в стартовой зоне
золото, если задумано как повторяемая точка
```

Поведение:

```text
игрок добывает
получает ресурс
объект временно исчезает
через cooldown появляется снова
```

Ответственный сервис:

```text
ResourceService
```

Начисление:

```text
только через CurrencyService
```

---

## 3. Ресурсы как награда за освоение зоны

Объект может дать ресурс, но при этом не быть обычным ресурсным узлом.

Пример:

```text
Игрок разобрал завал в ForestZone.
Получил +2 Wood.
Завал исчез навсегда.
```

Это не ResourceNode.

Это:

```text
ZoneClearObject
```

Ответственный сервис:

```text
WorldService
позже ZoneService
```

---

## 4. Что нельзя делать

Нельзя смешивать:

```text
камень как обычный ресурс с cooldown
и каменную кучу как объект очистки зоны
```

Если каменная куча нужна для прогресса зоны, она не должна респавниться.

---

## 5. Таблица типов

| Тип | Респавн | Даёт ресурс | Влияет на состояние зоны | Сохраняется |
|---|---:|---:|---:|---:|
| DecorOnly | Нет | Нет | Нет | Обычно нет |
| GlobalResourceNode | Да | Да | Нет | Обычно нет |
| ZoneClearObject | Нет | Может | Да | Да |
| ZoneResourceNode | Может | Да | Может зависеть от дизайна | Да/нет |
| ZoneOneTimeReward | Нет | Да/предмет | Может | Да |

---

## 6. Начисление ресурсов

Ресурсы нельзя менять напрямую:

```lua
profile.Wood += 1 -- плохо
```

Нужно использовать CurrencyService:

```lua
CurrencyService.AddWood(player, 1)
CurrencyService.AddStone(player, 1)
CurrencyService.AddMetal(player, 1)
CurrencyService.AddGold(player, 1)
```

---

## 7. Сохранение одноразовых объектов

Одноразовые объекты должны иметь стабильный ID.

Пример:

```text
ForestBlockage_1
ForestOldTree_1
ForestStonePile_1
```

После взаимодействия:

```lua
profile.ForestZoneClearedObjects[objectId] = true
```

При создании зоны:

```text
если objectId уже очищен — объект не создаётся
```

---

## 8. ForestZone: правила ресурсов

Для ближайшего этапа:

```text
ForestZone не должна создавать обычные респавнящиеся камни и деревья как часть очистки зоны.
```

Разрешено:

```text
ZoneClearObject даёт Wood/Stone один раз.
DecorOnly создаёт визуал.
ZoneResourceNode пока не добавлять или добавлять только явно.
```

---

## 9. Будущие лесные ресурсы

Позже можно добавить настоящие ресурсы лесной зоны:

```text
лоза
смола
грибы
травы
редкая древесина
```

Для каждого нужно заранее решить:

```text
респавнится ли он
как часто
влияет ли на состояние зоны
нужен ли ToolKitLevel
сохраняется ли индивидуально
```

---

## 10. Правило для Codex

В каждой задаче, где создаются ресурсы, явно указывать:

```text
Это обычный респавнящийся ресурс?
Это объект освоения зоны?
Это одноразовая награда?
Это декор?
```
