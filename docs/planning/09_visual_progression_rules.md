# 09. Visual Progression Rules

## Главный принцип

Игрок должен видеть результат своих действий не только в UI, но и в мире.

---

## Active

```text
густой лес
заросли
несколько препятствий
камни / старые деревья / завалы
табличка "Лесная зона"
```

---

## Empty

```text
меньше деревьев
несколько пней
очищенная площадка
тропинка
табличка "Лесная зона очищена"
нет старых ZoneClearObject
нет крупных деревьев Active-состояния
нет ForestZone resources от ResourceService
```

---

## Cleared

Будущее состояние полностью освоенной зоны.

```text
чистая площадка
более понятная тропинка
место под будущий NPC / объект / проход
табличка "Лесная зона освоена"
```

## Техническое правило

Визуал состояния зоны должен быть отделён от логики интерактивных объектов.

Желательная структура:

```text
ForestZone
ForestZoneDecor
ForestZoneInteractives
ForestZoneResources
ForestZoneSigns
ForestZoneVisualState
```

При переходе в Empty старые группы Active-состояния должны удаляться.
