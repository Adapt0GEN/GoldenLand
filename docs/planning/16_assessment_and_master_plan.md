# 16. Оценка проекта и мастер-план развития

Документ создан при подключении нового ведущего ассистента (Claude). Он фиксирует
независимую оценку текущего состояния и согласованный пошаговый план, чтобы
направление развития не терялось между сессиями.

---

## 1. Оценка текущего состояния

Проект — не заготовка, а рабочий вертикальный срез (~7300 строк Lua) с серверной
архитектурой из сервисов, сохранением через DataStore и дисциплиной документации.

### Сильные стороны

```text
сервер — источник истины, клиент только рисует
рабочий цикл: спавн → квест → ресурсы → участок → дом/склад/мастерская/кузница
прогрессия зон: ForestZone, RockZone
переработка металла: Metal → MetalIngot → MetalParts
сохранение и восстановление состояний между запусками
аккуратные docs и changelog
```

### Главные риски и разрывы

```text
1. Видение ≠ реализация: вижн про мультиплеер/острова/захват, MVP — одиночный.
2. Нет игрового «крючка»: нет боя, врагов, захвата территории.
3. Нет автоматизации/рабочих NPC.
4. PlotService — god-object (~2700 строк).
5. Весь мир строится кодом (тяжело итерировать визуал).
6. Не было инструментов качества (линтер/форматтер/типы) и автотестов.
7. Дрейф данных: Constants устарел, не было версии схемы профиля.
```

---

## 2. Зафиксированные стратегические решения

```text
Карта: гибрид — статичный визуал в .rbxl (Roblox Studio), спавн ресурсов/врагов/зон кодом.
Мультиплеер (план): несколько Places + TeleportService (остров игрока ↔ общие земли).
Бой (план): на старте простой — HP у врагов, удар, лут за зачистку.
```

### Фаза 1 — Игровой «крючок»: захват земли  [В РАБОТЕ]
Бой: оружие-Tool + серверный hitbox (R15). Охват: соло.
```text
- 1.1 основа боя: меч-Tool, серверный hitbox, враг-манекен с HP, лут, респавн
      [сделано в коде, нужен тест в Studio]
- 1.2 враждебный лагерь: кластер врагов + структура лагеря в отдельной зоне
      [сделано в коде, нужен тест в Studio]
- 1.3 захват: зачистка лагеря → зона помечается твоей → новый buildable-слот
```

### Фаза 2 — Автоматизация через NPC
```text
- спасаемый/нанимаемый NPC
- назначение NPC на здание → пассивная генерация ресурса во времени
- первый хаб автоматизации на отбитой земле
```

### Фаза 3 — Мультиплеерный фундамент (острова по приглашению)
```text
- несколько Places + TeleportService
- присоединение друзей на остров хозяина по приглашению
- базовые права гостей (строить/ломать)
```

### Фаза 4 — Общие земли: столица и групповые зоны
```text
- Place «Общие земли» + портал с острова
- столица как социально-торговый хаб
- первые общие PvE-зоны, заготовка под рейды
```

### Фаза 5 — Эндгейм (по существующим planning-докам)
```text
сытость/усталость → питомцы → Мрак → сооружения против Мрака →
гильдии → нейтральные земли → гильдейские завоевания (ресурсный sink)
```
